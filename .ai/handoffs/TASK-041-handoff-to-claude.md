# Handoff: TASK-041 -> Claude (Cross-Review)

## Summary of changes
Implemented a child-safe AI visual asset generation pipeline with explicit provider port, deterministic style-policy gates, moderation-before-preview/apply enforcement, and deterministic tool-call integration.

### New outbound port + adapter
1. `VisualGenerationPort`
- New outbound contract for provider-agnostic image generation payloads (`image_bytes`, `mime_type`, `provider_asset_id`, metadata).

2. `SafePresetVisualGenerationAdapter`
- Implements `VisualGenerationPort` with child-safe style presets.
- Defaults language and style to Polish-safe defaults.
- Returns deterministic PNG-like payloads (with PNG magic bytes) for moderation compatibility.

### New application service
3. `VisualAssetGenerationService`
- `request_preview(...)`:
  - validates request payload
  - enforces child-safe style policy
  - blocks photoreal-human generation in kid mode
  - moderates prompt text
  - generates image through `VisualGenerationPort`
  - moderates generated image before preview
  - caches moderated preview with deterministic `preview_id`
- `apply_preview(...)`:
  - re-runs image moderation before apply
  - persists asset through `AssetRepositoryPort`
- Emits `SafetyInterventionTriggered` for blocked paths.

### Deterministic tool-calling integration
4. `AIToolRegistry`
- Added `visual_generate` schema with explicit required args:
  - `project_id`, `world_id`, `prompt`, `style_preset`
- Unknown args disallowed for deterministic payload shape.

5. `RequestAICreationHelpService`
- Added `visual_generate` to permitted tool scopes (kid+parent).
- Added kid-mode validation gate for visual requests:
  - only safe style presets
  - photoreal-human request blocking
- Fixed validation flow so asset/image moderation and visual checks run regardless of registry-presence branch.
